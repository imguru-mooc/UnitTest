## 4. 프로덕션 코드 프로젝트 생성

### Step 4.1: 템플릿 선택

왼쪽 트리에서 다음 경로로 이동합니다:

```
Visual C#
  └── Windows 클래식 데스크톱
        └── 클래스 라이브러리(.NET Framework)  ← 선택!
```

| 필드 | 입력값 | 설명 |
|------|--------|------|

| 위치(L) | `C:\Projects\` | 저장 위치 (원하는 곳) |


### Step 13.2: public의 의미

```
┌─────────────────────────────────────────────────────────────────┐
│  접근 제한자 (Access Modifier)                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  public   : 어디서든 접근 가능 (다른 프로젝트에서도!)          │
│  internal : 같은 프로젝트 내에서만 접근 가능 (기본값)          │
│  private  : 같은 클래스 내에서만 접근 가능                     │
│                                                                 │
│  테스트 프로젝트에서 Calculator 클래스에 접근하려면             │
│  반드시 public이어야 합니다!                                    │
│                                                                 │
│  ┌─────────────────┐      ┌─────────────────┐                  │
│  │ Calculator.Tests │ ───→ │ Calculator      │                  │
│  │                 │      │ public class    │                  │
│  │                 │      │ ↑ public 필수!  │                  │
│  └─────────────────┘      └─────────────────┘                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 14. Add 메서드 작성

### Step 14.1: 메서드 시그니처 이해

```
┌─────────────────────────────────────────────────────────────────┐
│  메서드 시그니처                                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  public int Add(int a, int b)                                   │
│  ──┬─── ─┬─ ─┬─ ────┬────────                                   │
│    │     │   │      │                                           │
│    │     │   │      └─ 매개변수 (Parameters)                    │
│    │     │   │         두 개의 정수를 입력받음                  │
│    │     │   │                                                  │
│    │     │   └─ 메서드 이름 (Method Name)                       │
│    │     │                                                      │
│    │     └─ 반환 타입 (Return Type)                             │
│    │        정수(int)를 반환                                    │
│    │                                                            │
│    └─ 접근 제한자 (Access Modifier)                             │
│       어디서든 호출 가능                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Step 14.2: Add 메서드 작성

클래스 안에 Add 메서드를 추가합니다:

```csharp
using System;

namespace Calculator
{
    public class Calculator
    {
        public int Add(int a, int b)
        {
            return a + b;
        }
    }
}
```

### Step 14.3: 코드 분석

```csharp
public int Add(int a, int b)
{
    return a + b;
}
```

| 부분 | 설명 |
|------|------|
| `public` | 외부에서 호출 가능 |
| `int` | 반환 타입 (정수) |
| `Add` | 메서드 이름 |
| `int a` | 첫 번째 매개변수 |
| `int b` | 두 번째 매개변수 |
| `return a + b;` | 두 값을 더해서 반환 |

### Step 14.4: XML 주석 추가 (선택사항)

더 좋은 코드를 위해 XML 주석을 추가합니다:

```csharp
using System;

namespace Calculator
{
    /// <summary>
    /// 기본적인 사칙연산을 수행하는 계산기 클래스
    /// </summary>
    public class Calculator
    {
        /// <summary>
        /// 두 정수를 더합니다.
        /// </summary>
        /// <param name="a">첫 번째 숫자</param>
        /// <param name="b">두 번째 숫자</param>
        /// <returns>두 숫자의 합</returns>
        public int Add(int a, int b)
        {
            return a + b;
        }
    }
}
```

## 23. AAA 패턴 - Arrange 단계

### Step 23.1: AAA 패턴이란?

```
┌─────────────────────────────────────────────────────────────────┐
│  AAA 패턴 (Arrange-Act-Assert)                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  모든 단위 테스트의 기본 구조입니다.                           │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Arrange (준비)                                         │   │
│  │  • 테스트에 필요한 객체 생성                            │   │
│  │  • 테스트 데이터 준비                                   │   │
│  │  • 의존성 설정                                          │   │
│  └─────────────────────────────────────────────────────────┘   │
│                       ↓                                         │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Act (실행)                                             │   │
│  │  • 테스트하려는 동작 실행                               │   │
│  │  • 보통 한 줄의 코드                                    │   │
│  └─────────────────────────────────────────────────────────┘   │
│                       ↓                                         │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Assert (검증)                                          │   │
│  │  • 결과가 예상과 일치하는지 확인                        │   │
│  │  • Assert 메서드 사용                                   │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Step 23.2: Arrange 단계 작성

테스트에 필요한 객체를 생성합니다:

```csharp
[Test]

{
    // Arrange (준비)
    var calculator = new Calculator();
}
```

### Step 23.3: Arrange 단계 설명

```csharp
// Arrange (준비)

```

```

│  코드 분석                                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  var calculator = new Calculator();                             │
│  ─┬─ ────┬───── ─┬─ ────┬──────                                │
│   │      │       │      │                                      │
│   │      │       │      └─ 생성자 호출 (Calculator 인스턴스)   │

│   │      └─ 변수 이름 (calculator)                              │
│   │                                                             │
│   └─ var 키워드 (타입 추론, Calculator와 동일)                 │
```

## 24. AAA 패턴 - Act 단계

### Step 24.1: Act 단계란?

```
┌─────────────────────────────────────────────────────────────────┐
│  Act (실행) 단계                                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  • 테스트하려는 동작을 실행하는 단계                           │

│     │                 └─ 테스트 대상 메서드 호출                │
│     │                                                           │
│     └─ 결과를 저장할 변수                                      │
│                                                                 │
```

## 25. AAA 패턴 - Assert 단계

### Step 25.1: Assert 단계란?

```
┌─────────────────────────────────────────────────────────────────┐
│  Assert (검증) 단계                                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  • 실행 결과가 예상과 일치하는지 확인하는 단계                 │
│  • Assert 클래스의 메서드 사용                                 │
│  • 검증 실패 시 테스트 실패                                    │
│  • 검증 성공 시 테스트 통과                                    │
│                                                                 │
│  Assert.AreEqual(예상값, 실제값);                               │
│                    │       │                                    │
│                    │       └─ Act 단계에서 얻은 결과           │
│                    │                                            │
│                    └─ 우리가 기대하는 값                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

```csharp

    // Act (실행)
    int result = calculator.Add(2, 3);
    
    // Assert (검증)
    Assert.AreEqual(5, result);
}
```


## 47. Calculator.cs 최종 코드

```csharp
using System;

namespace Calculator
{
    /// <summary>
    /// 기본적인 사칙연산을 수행하는 계산기 클래스
    /// </summary>
    public class Calculator
    {
        /// <summary>
        /// 두 정수를 더합니다.
        /// </summary>
        /// <param name="a">첫 번째 숫자</param>
        /// <param name="b">두 번째 숫자</param>
        /// <returns>두 숫자의 합</returns>
        public int Add(int a, int b)
        {
            return a + b;
        }
        
        /// <summary>
        /// 두 정수를 뺍니다.
        /// </summary>
        /// <param name="a">첫 번째 숫자</param>
        /// <param name="b">두 번째 숫자</param>
        /// <returns>두 숫자의 차</returns>
        public int Subtract(int a, int b)
        {
            return a - b;
        }
        
        /// <summary>
        /// 두 정수를 곱합니다.
        /// </summary>
        /// <param name="a">첫 번째 숫자</param>
        /// <param name="b">두 번째 숫자</param>
        /// <returns>두 숫자의 곱</returns>
        public int Multiply(int a, int b)
        {
            return a * b;
        }
        
        /// <summary>
        /// 두 정수를 나눕니다.
        /// </summary>
        /// <param name="dividend">피제수 (나눠지는 수)</param>
        /// <param name="divisor">제수 (나누는 수)</param>
        /// <returns>나눗셈 결과</returns>
        /// <exception cref="DivideByZeroException">divisor가 0일 때</exception>
        public int Divide(int dividend, int divisor)
        {
            if (divisor == 0)
            {
                throw new DivideByZeroException("0으로 나눌 수 없습니다.");
            }
            
            return dividend / divisor;
        }
    }
}
```
